# 更新日志

## 2026-07-24 创作偏好档案,把主张沉淀成全书的"宪法"

- 架构页新增「创作偏好档案」:文风 / 禁忌 / 读者定位 / 其他主张四个字段,填一次,生成、重写、润色、大纲、架构每个环节都高优先级遵循——不用每次重复交代。
- 和 AI 研讨(架构研讨 / 重写研讨)聊出的主张,点「存进偏好档案」一键吸收,AI 会自动归类合并进对应字段,对话不再用完即弃。
- 档案随项目长期保存,正文怎么改都不影响它;它管的是"这本书该怎么写",不是某一章。

## 2026-07-24 校对结果也回显,打开即见

- 生成时自动修掉的硬伤会逐条留存,打开「编辑部 → 校对」直接看到修了哪些(原文 → 改后),不必再点一遍「开始校对」。
- 手动校对的结果同样留存:下次打开自动回填待修清单,勾一勾就能修。
- 与主审回显一致:正文一旦改动,旧校对清单自动失效,不会拿过期结果误导你。

## 2026-07-24 重写前先和 AI 聊聊,改得更准

- 重写章节时新增可选对话:说不清哪里不满意,就先和 AI 编辑聊几轮,它会把你的想法蒸馏成一条精准的修改意见,填进重写框再重写。
- 原来的快速重写照常保留:直接填意见或点优化方向 chip,一键重写,互不耽误。

## 2026-07-24 主审结果回显,不用重复审

- 生成时和手动审校的主审结果都会留存,打开「编辑部」直接回显最近一次评分,不必再点一遍「请主编审读」。
- 回显会标注来源(生成时审校 / 手动审校)、时间、回炉轮数与自动修复的硬伤数。
- 正文一旦改动(编辑 / 润色 / 重写 / 回滚),旧评分自动失效,避免对着新正文看旧分数。

## 2026-07-23 生成时审校把关 + 版本更新提醒

- 生成章节时自动跑「编辑部」流程:先校对、硬伤自动修掉,再请主审四维打分。
- 不达标会带着主审意见自动回炉重写,有上限(默认 3 轮),到点接受最好的一版——告别生成后手动「一直回炉」。
- 写作页新增审校配置:达标线(默认四维均 ≥7,可调到 8/9)、回炉上限、自动回炉开关。
- 生成结果卡与编辑部都会显示「达标 / 未达标」状态。
- 新增版本更新提醒:发现新版本时顶部提示刷新,并可查看本次更新内容。
